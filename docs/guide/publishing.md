# Publishing Guide

Use this starter as a clean public boundary.

## Before first push

1. Search for local paths, secrets, and product-specific names.
2. Replace placeholder maintainer information.
3. Decide whether the default MIT license matches your intent.
4. Verify the example task archive is safe to publish.

## Public repo health

Keep these files present and current:

- `README.md`
- `LICENSE`
- `CONTRIBUTING.md`
- `CODE_OF_CONDUCT.md`
- issue templates
- pull request template
