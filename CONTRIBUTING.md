# Contributing

Thanks for helping improve the workflow kit.

## Ground rules

- Keep changes small and explain the workflow problem being solved.
- Prefer repo-local, inspectable behavior over hidden automation.
- Do not add vendor lock-in unless it is clearly optional.
- Preserve the normalized archive schema when extending the runner.

## Typical contribution flow

1. Open an issue or reference an existing one.
2. Create a focused branch.
3. Add or update docs when behavior changes.
4. Include verification notes in the pull request.

## Good first contributions

- Improve README clarity.
- Add example archives.
- Add cross-platform command notes.
- Generalize skill templates without tying them to one tool.
